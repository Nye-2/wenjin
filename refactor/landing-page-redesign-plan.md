# 问津主页重设计计划（最终版）

> 基于三平面架构 + 用户痛点驱动
> 定稿：2026-04-29

---

## 一、用户痛点定位

问津的目标用户不是"想用 AI 写论文的人"，而是：

> **面对一个复杂学术写作任务，不知道从何开始、过程不可见、成果不可控的人。**

| 痛点 | 现有工具的问题 | 问津的解法 |
|------|---------------|-----------|
| 不知道从何开始 | ChatGPT 给一段文字，没有执行路径 | Chat Dock：对话确认后自动生成阶段计划 |
| 过程不可见 | 输入→等待→输出，中间黑盒 | Compute Stage：实时展示阶段、运行时块、文件、日志 |
| 成果不可控 | AI 直接覆盖原稿，改坏了无法回退 | WenjinPrism：diff review → apply/revert，版本化精修 |
| 工具割裂 | 文献管理、写作、排版各用各的 | 三平面在同一工作空间，数据不落地 |

**所有文案必须从痛点出发，而不是罗列功能。**

## 三、Hero

> **写论文不该是一场黑盒实验**
>
> 你说一句话，AI 规划路径、执行任务、生成初稿。全过程在你眼前展开，每一行修改你都能 review、确认、回退。

### 3.2 三平面标签（定稿）

```
Chat Dock          Compute Stage          WenjinPrism
  对话即启动          过程即透明             交付即版本
```

保留英文品牌名，下方小字说明核心价值。

### 3.3 产品预览

三个平面并排在浏览器窗口中，CSS 绘制微缩界面，带 subtle 动效：

- **Chat**：暖纸色 `#F7F4EE`，输入框有闪烁光标
- **Compute**：深色 `#0B0F14`，进度条有 shimmer
- **Prism**：纯白 `#FFFFFF`，diff 区域有 subtle 高亮

页面加载后，三个平面从下方依次滑入（stagger 0.15s）。

## 四、核心能力（3 张卡片）

每张卡片采用 **"痛点 → 解法 → 界面截图"** 三段式，纵向排列，箭头连接。

**卡片 1 — Chat Dock**
> 标题：**你说一句话，AI 理解一切**
>
> 痛点：面对一个大的写作任务，不知道从何开始，也不知道该给 AI 什么指令。
>
> 解法：用自然语言描述需求，AI 自动解析意图、确认参数、生成执行计划。不需要学习提示词。

**卡片 2 — Compute Stage**
> 标题：**任务执行过程，全部实时可见**
>
> 痛点：用 ChatGPT 写论文，输入一段文字，输出一段文字，中间发生了什么完全不知道。
>
> 解法：Compute Stage 自动展开，阶段进度、运行时块、子代理状态、sandbox 文件、执行日志——全部实时可见。

**卡片 3 — WenjinPrism**
> 标题：**AI 生成不直接覆盖原稿**
>
> 痛点：AI 改坏了你的论文，无法回退，只能手动恢复。
>
> 解法：AI 生成的内容进入待确认队列。你在 diff 视图中逐条 review，选择 apply、discard 或 revert。主稿工程始终可控。

**视觉**：clean white card + subtle shadow + rounded-2xl，左侧文字右侧截图（40%）。

## 五、产品演示

### 5.1 场景：学位论文开题

选择"开题"而非"文献综述"——开题是研究生最焦虑的节点，更能体现三平面协作的价值。

### 5.2 三步骤

**Step 1 — 对话中发起**
> 用户："我要开题了，方向是多模态学习，帮我做一份开题研究计划"
>
> AI 回复确认卡片：意图「深度研究」、参数确认、5 个阶段预览。用户点击「开始执行」。

**Step 2 — 工作台中观察**
> Compute Stage 展开，Phase Timeline 显示阶段进度，Runtime Blocks 实时更新检索状态，Sandbox 出现文献综述文件。

**Step 3 — 主稿台中确认**
> 任务完成后，WenjinPrism 展示 diff preview，用户 review 后 apply 进入主稿。后续修改再次触发待确认队列。

### 5.3 视觉

横向时间线连接 3 个 step，每个 step 配真实产品截图，滚动时依次高亮。底部 CTA："体验完整工作流 →"。

## 七、视觉规范

### 7.1 颜色

| 区域 | 背景 | 说明 |
|------|------|------|
| Hero | `#F4F0E8` | 产品预览是视觉焦点 |
| 核心能力 | `#FBF8F2` | 轻微提升，与 Hero 区分 |
| 产品演示 | `#F4F0E8` | 回到基础色，形成节奏 |
| Types + CTA | `#FBF8F2` | 最终提升，引导行动 |

**禁用 route-grid 和 route-topography**，背景保持干净。

### 7.2 卡片

- **禁用 LiquidGlassCard**
- **统一 clean card**：白色背景 + subtle shadow + rounded-2xl
- 只有产品预览组件可用特殊样式

### 7.3 圆角

- Section：无圆角
- 卡片 / 按钮：rounded-2xl（16px）
- Badges / Pills：rounded-full

### 7.4 间距

- Section 之间：py-20（80px）
- 卡片内部：p-6（24px）
- 卡片之间：gap-5（20px）
- 文字块之间：mt-4 / mt-6
