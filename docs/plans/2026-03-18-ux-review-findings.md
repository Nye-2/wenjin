# UX Review Findings - Pre-Launch Audit

**Date:** 2026-03-18
**Type:** UX Review Report
**Scope:** Pre-launch user experience audit for 5 workspace types

---

## Executive Summary

本报告记录了上线前用户体验审计发现的所有问题，按严重程度分类，并提供修复优先级建议。

**总计发现:** 58 个问题
- P0 (阻断上线): 5 个
- P1 (强烈建议): 22 个
- P2 (建议优化): 31 个

---

## P0 Issues (阻断上线 - 必须修复)

### P0-1: 导出功能按钮为占位符

**位置:** `compile-export/page.tsx`
**问题:** 导出格式按钮（PDF, Word, LaTeX, Markdown）没有实际功能
**影响:** 用户点击后无反应，造成困惑
**建议:**
- 实现真实的导出功能
- 或者暂时移除这些按钮

### P0-2: 图表生成章节关联硬编码

**位置:** `figure-generation/page.tsx`
**问题:** 章节关联下拉框硬编码 "第一章" 到 "第四章"
**影响:** 当大纲有5+章时无法正确关联
**建议:** 从 thesis_writing 的 artifact 动态获取章节列表

### P0-3: 添加文献按钮无功能

**位置:** `literature/page.tsx`
**问题:** "添加文献"按钮没有 onClick 处理器
**影响:** 用户无法添加文献
**建议:** 实现添加文献功能或显示 "即将推出"提示

### P0-4: 筛选按钮无功能

**位置:** `literature/page.tsx`
**问题:** "筛选"按钮没有 onClick 处理器
**影响:** 用户无法筛选文献列表
**建议:** 实现筛选功能或移除按钮

### P0-5: 积分系统提示处理

**位置:** `literature/page.tsx`
**问题:** "智能盘点（20积分）" 未解释积分系统
**影响:** 用户不知道积分从哪来、怎么用
**建议:**
- 添加积分系统说明或 Tooltip
- 或暂时移除积分显示

## P1 Issues (强烈建议修复)

### P1-1: 语言混合

**位置:** `workspaces/[id]/page.tsx`
**问题:** `workspaceTypeLabels` 英文，其他地方中文
**影响:** 视觉不一致，**建议:** 统一为中文

### P1-3: 任务执行无进度指示

**位置:** 所有功能页面
**问题:** 长时间任务只显示"任务执行中..."，没有进度条或**影响:** 用户不知道执行到哪一步了
**建议:** 添加进度指示（如：1/3, 2/3... 或百分比进度条）

### P1-10: Artifact ID 选择器

**位置:** `sci/writing/page.tsx`
**问题:** 上下文 Artifact ID 需要手动输入
**影响:** 用户不知道 artifact ID，难以使用
**建议:**
- 提供下拉选择器
- 或显示最近 artifact 列表

### P1-12: Workspace 计数不准确

**位置:** `workspaces/page.tsx`
**问题:** `paperCount` 和 `artifactCount` 瀏览为 0
**影响:** 显示不真实
**建议:** 从 API 获取真实数据

### P1-4: 返回按钮行为

**位置:** 所有功能页面
**问题:** 返回按钮返回 workspace 主页
**影响:** 无法返回上一页
**建议:**
- 改为返回上一页
- 或添加面包屑导航

### P1-5: Step 禁用状态提示

**位置:** `thesis-writing/page.tsx`
**问题:** Step 2 禁用时只有 opacity 双没有说明
**影响:** 用户可能不清楚为什么不能点击
**建议:** 添加 Tooltip 说明"请先生成大纲"

### P1-6: 无历史记录展示

**位置:** 所有功能页面
**问题:** 除了 thesis_writing 外，功能页面不显示历史执行记录
**影响:** 用户不知道之前执行过什么
**建议:** 在主区域添加"执行历史"区块

### P1-7: 主区域空状态

**位置:** 多个功能页面
**问题:** deep-research, literature-search, background-research 等页面主区域是空状态占位
**影响:** 用户不知道页面是做什么的
**建议:** 显示最近产出或引导信息
