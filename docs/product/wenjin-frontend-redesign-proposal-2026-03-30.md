# 问津 Wenjin 前端重塑方案

更新时间: 2026-03-30
适用范围: `frontend/` 品牌、首页、认证、workspace 列表、workspace workbench
当前状态: 提案，未实施

## 1. 目标

将当前项目前端品牌与界面，重塑为“问津 / Wenjin”体系。

这次改版不只是改名，而是同步完成三件事:

1. 把产品定位从“学术写作助手”升级为“面向论文、申报、专利与项目写作的 AI 工作台”。
2. 把首页、认证页、工作空间、工作台的叙事语言统一为“路径、推进、交付”。
3. 把视觉母题切换为“问路、问津、寻径、抵达”。

## 2. 品牌基线

### 2.1 品牌名

- 中文主名: `问津`
- 英文名: `Wenjin`

### 2.2 Hero 母句

- 中文: `向研究深处问津`
- 英文: `Find the way, move the work forward.`

### 2.3 产品定位

`面向论文、申报、专利与项目写作的 AI 工作台`

### 2.4 品牌语气

不再强调“诗性空灵”和“站在浪潮之巅”，改为强调:

- 找到路径
- 看清阶段
- 推进工作
- 留下成果

语气要求:

- 克制
- 清晰
- 有方向感
- 有执行感
- 不喊口号，不做泛 AI 叙事

## 3. 现状判断

当前前端存在三类明显错位:

1. 品牌是偏东方意象，但产品文案仍偏“academic assistant”。
2. 首页是情绪化 hero，工作区是功能化界面，中间缺少统一的工作流叙事。
3. 视觉语言以“波浪 + glass morphism”为主，更像品牌展示页，不够像严肃生产工具。

现有结构中仍可复用的部分:

- workspace + chat 的整体产品骨架
- 功能入口已按 workspace 类型组织
- 组件拆分足够细，适合渐进改造

不建议继续沿用的部分:

- 旧的 wave / crest / foam 命名体系
- 首页过重的波浪装饰和空泛哲学段落
- 侧边栏里的“聊天历史即主结构”表达

## 4. 新视觉方向

## 4.1 关键词

- Survey map
- Route line
- Field note
- Task ledger
- Editorial workspace

## 4.2 视觉母题

“问津”不再用大面积海浪，而是改用:

- 路线图
- 路径节点
- 水文等高线
- 标注卡片
- 纸面与仪表感

一句话概括:

`像一张被不断标注、不断推进的研究航线图，而不是一块会发光的玻璃面板。`

## 4.3 色彩建议

保留蓝系信任感，但从“水波渐变”改为“深蓝 + 青绿 + 黄铜 + 纸白”。

建议色板:

- `--brand-ink`: `#132235`
- `--brand-navy`: `#1F4263`
- `--brand-teal`: `#2E6F6D`
- `--brand-cyan`: `#5C97A5`
- `--brand-brass`: `#A67C39`
- `--bg-paper`: `#F7F4EE`
- `--bg-wash`: `#EEF2F3`
- `--line-soft`: `#D7DEE2`

原则:

- 弱化高亮蓝
- 增加纸面感
- 强化分层边界
- CTA 用黄铜或深青，而不是继续用“蓝色发光按钮”

## 4.4 字体建议

- 中文标题: `Noto Serif SC` 或 `Source Han Serif SC`
- 中文正文字体: `Noto Sans SC`
- 英文显示字体: `Instrument Serif` 或 `Cormorant Garamond`
- 英文正文: `DM Sans`

原则:

- 英文名 `Wenjin` 需要更有识别度
- 中文正文保持工具属性，避免全站“书法感”

## 4.5 动效建议

- 路径线条缓慢展开
- 卡片按“节点”顺序淡入
- 当前任务使用脉冲点而不是大面积 shimmer
- 页面转场以横向推进或上推为主

避免:

- 大面积水波飘动
- 过多 glass blur
- 无意义 hover 漂浮

## 5. 信息架构重塑

## 5.1 首页定位

首页不再是“品牌展示页”，而是“工作台入口页”。

首页必须在 1 屏内讲清楚:

1. 这是做什么的
2. 适合什么任务
3. 工作如何被推进
4. 用户下一步该点哪里

## 5.2 workspaces 页定位

从“卡片列表”改为“任务入口总览”。

应该突出:

- 最近推进中的工作
- workspace 类型模板
- 最近产出
- 一键继续上次任务

## 5.3 workbench 页定位

从“聊天页 + 若干面板”改为“研究驾驶舱”。

聊天仍是主入口，但不是唯一中心。核心结构应变为:

- 左侧: 路径导航
- 中间: 当前任务/对话主线
- 右侧: 来源、产出、状态

## 6. 首页改版方案

## 6.1 Hero 结构

建议用 2 栏 Hero，而不是当前纯居中海报式布局。

左侧:

- `问津`
- `Wenjin`
- 母句
- 定位句
- 说明段落
- 双 CTA

右侧:

- “工作流路线图”卡片
- 5 个阶段节点
- 当前活跃状态示意
- 支持的任务类型标签

文案建议:

```text
问津
Wenjin

向研究深处问津
面向论文、申报、专利与项目写作的 AI 工作台
Find the way, move the work forward.

从选题调研到成稿交付，在一个工作空间里串起来源、推理、写作、修订与成果沉淀。

[进入工作空间] [查看能力]
```

## 6.2 Hero 右侧示意卡

标题:

`当前工作路径`

节点:

1. `选题与背景`
2. `文献与证据`
3. `结构与论证`
4. `写作与修订`
5. `评审与交付`

每个节点展示:

- 状态
- 对应产物
- 最近动作

这张卡要让用户一眼理解:

`Wenjin 不是聊天机器人，而是一个会把工作推进下去的路线化工作台。`

## 6.3 首页第二屏

改为“支持的任务类型”，不再展示抽象 feature icon。

建议四个入口卡:

- `论文写作`
- `项目申报`
- `专利撰写`
- `研究工作流`

每张卡应包含:

- 适用场景
- 典型产出
- 一条具体价值

## 6.4 首页第三屏

改为“如何推进一项复杂任务”。

结构:

1. 收集来源
2. 形成判断
3. 生成草稿
4. 修订与评审
5. 沉淀成果

用横向时间线或竖向路线图表现，不再使用哲学引用卡片。

## 6.5 首页底部 CTA

建议:

- 标题: `把下一项研究任务放进一条可推进的工作流`
- 文案: `创建 workspace，开始组织来源、推进写作，并把成果沉淀下来。`
- 按钮: `进入工作空间`

## 7. 认证页改版方案

## 7.1 目标

认证页不再强调“欢迎回来”，而是强调“继续一条工作路径”。

## 7.2 布局

保留左右分栏，但左侧从品牌陈列改为“路径概览”。

左侧文案:

- 标题: `向研究深处问津`
- 描述: `一个工作空间，连接来源、判断、草稿与成果。`

左侧三张卡:

- `来源入库`
- `结构成稿`
- `评审交付`

右侧表单:

- 登录: `进入你的工作空间`
- 注册: `创建你的问津账户`

## 7.3 交互

- 登录 / 注册切换保留 pill tab
- 验证码发送按钮改为更工具化样式
- 表单底部增加“支持任务类型”小标签

## 8. Workspaces 页改版方案

## 8.1 当前问题

当前是标准 SaaS 卡片墙，功能正确，但缺少“继续推进”的感觉。

## 8.2 新结构

顶部:

- 页面标题: `我的工作路径`
- 副标题: `继续推进正在进行的论文、申报、专利与研究任务`
- 主按钮: `新建 workspace`

中部第一块:

- `继续最近任务`
- 展示 3 个最近活跃 workspace
- 每个显示:
  - 当前阶段
  - 最近产出
  - 最后更新时间
  - 继续按钮

中部第二块:

- `按类型开始`
- 论文 / 申报 / 专利 / 软件著作权 / 学位论文

中部第三块:

- `全部 workspace`
- 保留搜索和筛选

## 8.3 卡片表达

不再只显示“paper count / artifact count”，增加:

- 当前阶段
- 最近动作
- 目标产物

例:

```text
国家自然科学基金项目申报
类型: 项目申报
当前阶段: 实验设计
最近动作: 完成背景调研摘要
目标产物: 申报书初稿
```

## 9. Workbench 改版方案

## 9.1 总体定位

改为“研究驾驶舱”，不再让用户觉得自己进入的是“一个聊天页面”。

## 9.2 三栏结构

左栏: `路径栏`

- workspace 基本信息
- 当前阶段导航
- 会话列表
- 快速入口

中栏: `主工作区`

- 当前任务头部
- 聊天 / 指令 / 草稿主内容
- 快速启动卡
- 任务流反馈

右栏: `证据与成果`

- 文献
- artifact
- 最近生成结果
- 状态摘要

## 9.3 顶部 Header

增加统一工作头部:

- workspace 名称
- 类型 badge
- 当前阶段
- 最近任务状态
- 主 CTA: `继续当前任务`

## 9.4 左侧路径栏

现有 `AppShellSidebar` 应从“导航栏”改为“路线栏”。

建议分四段:

1. 当前 workspace 概况
2. 阶段路径
3. 会话与草稿
4. 返回全部 workspace

阶段路径建议:

- 背景调研
- 资料收集
- 结构设计
- 写作修订
- 评审交付

当前阶段高亮，已完成阶段显示实心标记。

## 9.5 中央主工作区

聊天页顶部增加任务上下文条:

- 当前目标
- 推荐动作
- 已选 skill / model
- 当前可生成的产物

默认空状态不要再写“Start Your Research Journey”，改为:

`从当前阶段开始，告诉问津你要推进什么。`

并给出三个操作卡:

- `生成研究路径`
- `继续上次草稿`
- `查看当前证据`

## 9.6 右侧证据与成果栏

统一成一个可切换的 inspector:

- Sources
- Drafts
- Outputs
- Activity

这比现在分散的面板更像工作台。

## 9.7 移动端

移动端要从“三栏”降为“单栏 + 底部切换”:

- 主内容
- 路径
- 证据

不要在移动端保留复杂侧边栏折叠逻辑。

- `frontend/app/page.tsx`

- `frontend/components/workspace/TaskSummaryStrip.tsx`
- `frontend/components/workspace/WorkspaceResultPanel.tsx`
- `frontend/components/workspace/ArtifactLibrary.tsx`

## 10.3 设计 token 重构

当前 `globals.css` 里的旧品牌 token 建议两步走:

第一步:

- 保留现有 token
- 增加通用别名 `--brand-*`
- 新组件优先使用 `--brand-*`

第二步:

- 清理旧品牌 token
- 统一到 `--brand-*` / `--surface-*` / `--text-*`

这样可以降低一次性重命名带来的风险。

## 11.1 首页

- 定位句: `面向论文、申报、专利与项目写作的 AI 工作台`

## 11.2 登录

- 标题: `进入你的工作空间`
- 描述: `继续推进来源整理、结构设计、写作修订与成果交付。`

## 11.3 注册

- 标题: `创建你的问津账户`
- 描述: `从第一份来源开始，建立一条可持续推进的工作路径。`

## 11.4 workspace 列表

- 标题: `我的工作路径`
- 描述: `继续正在进行的论文、申报、专利与研究任务。`

## 12. 分阶段实施建议

### Phase 1: 品牌与首页

- 品牌名替换为 `问津 Wenjin`
- 首页 Hero 重做
- metadata / 登录 / README 同步

### Phase 2: 认证与 workspace 列表

- auth shell 改版
- workspace list 改为“继续任务 + 模板入口”

### Phase 3: workbench 骨架

- sidebar 改为路线栏
- workbench header 增加阶段与目标
- 右栏 inspector 收敛

### Phase 4: 细节统一

- token 清理
- 图标与插画统一
- 中英文文案补齐
- 动效与响应式收尾

## 13. 推荐实施顺序

如果要尽快看到“品牌完全换掉”的效果，建议按以下顺序动手:

1. 首页
2. 顶栏与登录注册
3. workspace 列表
4. workbench 左栏与顶部
5. chat 空状态与右栏

原因:

- 首页 + 认证先改，品牌感立即成立
- workspace 列表改完，产品入口逻辑就变了
- 最后再改 workbench，可以避免一次性大面积回归

## 14. 结论

“问津”这套方案最适合把当前产品从“学术助手”拉到“复杂知识工作工作台”。

这次前端重塑最关键的不是只替换品牌字符串，而是完成三件事情:

1. 从“展示品牌意境”转为“展示工作路径”
2. 从“功能堆叠”转为“阶段推进”
3. 从“聊天入口”转为“工作台入口”

如果按本方案实施，最终用户看到的会是:

`一个能把复杂研究任务持续往前推进的 AI 工作台，而不是一个会聊天的学术网页。`
